Dynamics 365 / Dataverse Account Duplicate Cleanup Scripts
==========================================================

Purpose:
--------
These scripts help clean duplicate Account records in Dynamics 365 / Dataverse.

The dying Accounts are identified by a custom Yes/No flag:
    meta_tobecleaned = 1

The master Account is identified by matching:
    1. ABL Number
    2. OOB Account Address fields

Custom field assumptions:
-------------------------
    meta_tobecleaned  -- Yes/No field on Account. 1 = dying / to be cleaned.
    meta_ablnumber    -- ABL number field on Account.

OOB address fields used:
------------------------
    address1_line1
    address1_line2
    address1_line3
    address1_city
    address1_stateorprovince
    address1_postalcode
    address1_country

Important Notes:
----------------
1. Do NOT run these scripts directly against the Dataverse SQL/TDS endpoint from SSMS.
   The Dataverse SQL/TDS endpoint is read-only.

2. Use SQL 4 CDS in XrmToolBox or convert this logic into a C# console app using Dataverse SDK/Web API.

3. Test first in DEV or UAT.

4. Export/back up records before running in production.

5. Run preview scripts first.

6. Only records with exactly one matching master Account should be moved.

7. Records with multiple possible master matches should be reviewed manually.

8. Records with no master match should not be moved.

9. The scripts below are written for Account-to-Account cleanup only.

10. Since SQL CTEs only apply to the one statement immediately following them, each update script repeats the same AccountMap logic.


================================================================================
SCRIPT 1: Preview Dying Account to Master Account Mapping
================================================================================

Purpose:
--------
Shows all dying Accounts where meta_tobecleaned = 1 and their possible master Accounts,
matched using ABL number and OOB address fields.

Use this script first to review the proposed Account mapping.

SQL:
----

WITH NormalizedAccounts AS
(
    SELECT
        accountid,
        name,
        statecode,
        meta_tobecleaned,
        UPPER(LTRIM(RTRIM(ISNULL(meta_ablnumber, '')))) AS ABLKey,
        UPPER(LTRIM(RTRIM(ISNULL(address1_line1, '')))) AS Line1Key,
        UPPER(LTRIM(RTRIM(ISNULL(address1_line2, '')))) AS Line2Key,
        UPPER(LTRIM(RTRIM(ISNULL(address1_line3, '')))) AS Line3Key,
        UPPER(LTRIM(RTRIM(ISNULL(address1_city, '')))) AS CityKey,
        UPPER(LTRIM(RTRIM(ISNULL(address1_stateorprovince, '')))) AS StateKey,
        UPPER(REPLACE(REPLACE(LTRIM(RTRIM(ISNULL(address1_postalcode, ''))), ' ', ''), '-', '')) AS PostalKey,
        UPPER(LTRIM(RTRIM(ISNULL(address1_country, '')))) AS CountryKey
    FROM account
),
AccountMapRaw AS
(
    SELECT
        dying.accountid AS DyingAccountId,
        dying.name AS DyingAccountName,
        master.accountid AS MasterAccountId,
        master.name AS MasterAccountName,
        dying.ABLKey AS ABLNumber,
        dying.Line1Key,
        dying.Line2Key,
        dying.Line3Key,
        dying.CityKey,
        dying.StateKey,
        dying.PostalKey,
        dying.CountryKey
    FROM NormalizedAccounts dying
    INNER JOIN NormalizedAccounts master
        ON dying.ABLKey = master.ABLKey
        AND dying.Line1Key = master.Line1Key
        AND dying.Line2Key = master.Line2Key
        AND dying.Line3Key = master.Line3Key
        AND dying.CityKey = master.CityKey
        AND dying.StateKey = master.StateKey
        AND dying.PostalKey = master.PostalKey
        AND dying.CountryKey = master.CountryKey
        AND dying.accountid <> master.accountid
    WHERE
        dying.meta_tobecleaned = 1
        AND ISNULL(master.meta_tobecleaned, 0) = 0
        AND dying.statecode = 0
        AND master.statecode = 0
        AND dying.ABLKey <> ''
        AND dying.Line1Key <> ''
        AND dying.CityKey <> ''
        AND dying.StateKey <> ''
        AND dying.PostalKey <> ''
)

SELECT *
FROM AccountMapRaw
ORDER BY DyingAccountName;

--------

Do NOT move child records for these Accounts until they are manually reviewed.

        UPPER(REPLACE(REPLACE(LTRIM(RTRIM(ISNULL(address1_postalcode, ''))), ' ', ''), '-', '')) AS PostalKey,

    DyingAccountId,
    DyingAccountName,

    StateKey,
    PostalKey
HAVING COUNT(*) > 1

SCRIPT 3: Find Dying Accounts with No Master Match
================================================================================

Purpose:
--------
Finds Accounts marked as meta_tobecleaned = 1 that do not have a matching master Account.

These records should not be moved until the data is corrected or a master is manually selected.

SQL:
----

WITH NormalizedAccounts AS
(
    SELECT
        accountid,
        name,
        statecode,
        meta_tobecleaned,
        UPPER(LTRIM(RTRIM(ISNULL(meta_ablnumber, '')))) AS ABLKey,
        UPPER(LTRIM(RTRIM(ISNULL(address1_line1, '')))) AS Line1Key,
        UPPER(LTRIM(RTRIM(ISNULL(address1_line2, '')))) AS Line2Key,
        UPPER(LTRIM(RTRIM(ISNULL(address1_line3, '')))) AS Line3Key,
        UPPER(LTRIM(RTRIM(ISNULL(address1_city, '')))) AS CityKey,
        UPPER(LTRIM(RTRIM(ISNULL(address1_stateorprovince, '')))) AS StateKey,
        UPPER(REPLACE(REPLACE(LTRIM(RTRIM(ISNULL(address1_postalcode, ''))), ' ', ''), '-', '')) AS PostalKey,
        UPPER(LTRIM(RTRIM(ISNULL(address1_country, '')))) AS CountryKey
    FROM account
),
MatchedDyingAccounts AS
(
    SELECT DISTINCT
        dying.accountid AS DyingAccountId
    FROM NormalizedAccounts dying
    INNER JOIN NormalizedAccounts master
        ON dying.ABLKey = master.ABLKey
        AND dying.Line1Key = master.Line1Key
        AND dying.Line2Key = master.Line2Key
        AND dying.Line3Key = master.Line3Key
        AND dying.CityKey = master.CityKey
        AND dying.StateKey = master.StateKey
        AND dying.PostalKey = master.PostalKey
        AND dying.CountryKey = master.CountryKey
        AND dying.accountid <> master.accountid
    WHERE
        dying.meta_tobecleaned = 1
        AND ISNULL(master.meta_tobecleaned, 0) = 0
        AND dying.statecode = 0
        AND master.statecode = 0
        AND dying.ABLKey <> ''
        AND dying.Line1Key <> ''
        AND dying.CityKey <> ''
        AND dying.StateKey <> ''
        AND dying.PostalKey <> ''
)

SELECT
    a.accountid,
    a.name,
    a.meta_ablnumber,
    a.address1_line1,
    a.address1_line2,
    a.address1_line3,
    a.address1_city,
    a.address1_stateorprovince,
    a.address1_postalcode,
    a.address1_country
FROM account a
LEFT JOIN MatchedDyingAccounts m
    ON a.accountid = m.DyingAccountId
WHERE
    a.meta_tobecleaned = 1
    AND a.statecode = 0
    AND m.DyingAccountId IS NULL
ORDER BY a.name;

Recommended Production Approach
================================================================================

For production, a safer approach is to create a custom Dataverse mapping table first.

Suggested table:
----------------
    meta_accountcleanupmap

Suggested columns:
------------------
    meta_dyingaccount       Lookup to Account
    meta_masteraccount      Lookup to Account
    meta_ablnumber          Text
    meta_matchstatus        Choice: Pending, Approved, Ambiguous, No Match, Processed, Failed
    meta_childrecordsmoved  Yes/No
    meta_processedon        Date/Time
    meta_error              Multiline text

Recommended flow:
-----------------
1. Populate the mapping table using the matching logic.
2. Review and approve the mappings.
3. Move child records only for approved mappings.
4. Validate child record counts.
5. Deactivate dying Accounts.
6. Keep the mapping table as an audit trail.
